# 🎮 ComputerNetwork-TEAM-10

## TCP/UDP Socket 기반 콘솔 오목 게임

본 프로젝트는 **Computer Network 수업 팀 프로젝트**로,  
**TCP/UDP 소켓 통신을 이용하여 온라인 오목 게임을 구현**한 프로젝트입니다.  
클라이언트는 콘솔 환경에서 오목을 플레이하며, 서버는 여러 클라이언트의 접속 관리, 게임 진행,  
돌 배치 검증 및 승리 판정을 담당합니다.

---

## 📌 프로젝트 개요

-   **주제:** TCP/UDP 소켓 기반 네트워크 오목 게임
-   **개발 언어:** Python
-   **구현 형태:** 콘솔 기반 UI
-   **구성:**
    -   온라인 오목(2인 대전)
    -   TCP 기반 통신
    -   서버 주도형 게임 진행
    -   턴 관리 / 돌 배치 검증 / 5목 승리 판정

---

---

## 📌 역할분담

-   **서명준:**서버
-   **김우진:**ui/로직
-   **권진욱:**클라이언트
-   **Sayed Refat:**로직

---

## 🚀 주요 기능

### 서버 기능

-   다중 클라이언트 접속 관리 (대기 로비)
-   플레이어 매칭 및 게임 시작
-   각 클라이언트의 좌표 입력 수신
-   돌의 유효성 검사 및 보드 업데이트
-   5목(가로/세로/대각선) 승리 판정
-   게임 턴 관리 및 브로드캐스트

---

### 클라이언트 기능

-   TCP/UDP 방식으로 서버와 통신
-   콘솔 화면에서 오목판 렌더링
-   좌표 입력(UI)
-   서버 지시에 따른 턴 진행
-   승패 결과 출력
-   간단한 채팅

---

### 실행 방법

- python Server.py
- python Client.py
    - ip 입력
    - 플레이어 이름 입력
    - 상대 매칭
- restart는 서로 동의해야 시작

---

### 진행 과정

- 서버는 TCP소켓 기반으로 HTTP를 수동 피싱하여 JSON 응답을 보내주며, OmokGame 인스턴트로 게임의 정보를 관리합니다. 클라이언트는 pygame UI를 통해 주기적으로 상태를 폴링하고, 채팅,재시작,턴 등 정보를 HTTP POST로 보냅니다. 플레이어는 방에 
들어오는 순서대로 BLACK/WHITE를 한자리씩 배정받고, 두 플레이어가 모두 들어와야 게임이 시작됩니다. 만약 두 플레이어가 이미 모두 들어온 상태라면 이후 들어오는 플레이어는 관전자가 됩니다. 게임종료후 두플레이어 모두 RESTART 버튼을 눌러야 재시작됩니다.
